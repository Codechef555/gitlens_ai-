# GitLens AI — 3-minute demo script

## 0:00–0:20 — Problem

“Joining an unfamiliar repository is expensive. Engineers lose days tracing architecture, dependencies, and hidden risk. GitLens AI makes that context available immediately.”

Show the landing page and tagline.

## 0:20–0:45 — Import

Click **Import repository**, paste a public GitHub URL, and show the validation/cloning progress. Explain that GitLens reads the codebase and creates structured repository context.

## 0:45–1:15 — Architecture

Open the workspace. Pan and zoom the architecture map, select a node, and call out its relationships and representative files.

## 1:15–1:45 — Dependencies

Switch to **Dependencies**. Search for a file, click it, and highlight imports and importers. Toggle circular and unused-module filters.

## 1:45–2:15 — Change impact

Open **Change impact**. Ask, “What happens if I rename UserService?” Explain the affected files, risk score, API impact, suggested tests, and reasoning.

## 2:15–2:40 — Risks and docs

Open **Risks & docs**. Show prioritized findings, then switch generated documentation tabs and export a Markdown file.

## 2:40–3:00 — Repository chat and close

Open Chat. Ask “Where is JWT verified?” Show a streamed, source-grounded response. Close: “GitLens AI turns any codebase into a shared mental model so teams can ship with confidence.”
